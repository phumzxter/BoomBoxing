Implementation

Game architecture

Boom Boxing is developed by Unity. We also using Github to port for any code changing. This game using provided component-based architecture from Unity.

As implemented in the figure, each game object is attract to multiple components to setup its size (Renderer), how it transform by actions from the input (Transform), its hitbox(Rigid Body), where is it standing in between other object and last of all its behavior based on the mechanics we defined before (Mono Behavior).

There are three main game objects in this game: the environment, the avatar or characters and HUD.

For environment and HUD, we try to make it as simple, as friendly as possible.

Character & Game design

Actions

The avatar in Boom Boxing is design to react perfectly with the input from player. Each state of action is bind with a boolean variable, so each command is input, this variable will trigger the animation, also the transform (position) of this avatar and different part of its body. For example: moving forward will move the character up along with the animation of the legs moving. Input of our game is one-time per command so the game have to pre-determine the distance of moving. It’s also the same to other attack action.

To translate the voice of player to actions, Boom boxing using Speech API from Unity library, inherited from the C# library. This API already have trained model for English (U.S.), English (U.K.), French, German, Japanese, Mandarin Chinese, and Spanish. For this game, we only using English. In detail, we using Keyword Recognizer as main component.

The implementation is very straight-forward, if the speech model recognizes the word from microphone, it will automatically trigger moving or attack.

Hitbox

The avatar will have separate hitbox for each section part of the body.

It has hitboxs for upper body and bellow to trigger the damage take from other avatar. For hand and feet, we set it as a sphere for easier contact of attack actions. The last one is forearm hitbox, if attack hitbox contact to this part while avatar is on defense state, damage won’t be count.

Bot

Every component from main avatar, we apply it for the bot except the actions controller. For movement, the bot will always face up to the other avatar, wherever the position of other avatar is, even move left or right. Also, the bot will try to move forward, and attack the player.
